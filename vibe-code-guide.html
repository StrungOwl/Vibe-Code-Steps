Step 7

Let Your Agent Build

Best practices for prompting your agent.

Pseudo Code

A simplified, human-readable description of what your program should do — written in plain language. Think through logic without worrying about syntax.

"When the user clicks the canvas, generate a new shape at that position. Each shape should inherit the color of the previous one but rotate 15 degrees."

Be Descriptive

The more detail you give, the better the output. Instead of "make it look nice," say: "Use a dark background (#0a0a0a), thin white lines, and animate them with a sine wave that oscillates every 3 seconds."

Break into components: layout, interaction, animation, color, typography. Describe each one separately.

Think Like a Composer

Every musician in an orchestra has a strength. Treat each chat the same way — one for planning, one for building, one for troubleshooting, one for editing, one for critique. Don't cross-pollinate.

UI Testing with Playwright CLI

Playwright CLI lets your agent open a real browser, interact with your UI, and verify it works — all without you lifting a finger. Describe what to test and let Claude run it.

Model Selection

Step 7 — Continued

Let Your Agent Build

Useful /commands to know about

/model

Switch models mid-session. Match the model to the job:

  • Opus — complex reasoning, architecture, hard bugs
  • Sonnet — everyday building, writing, iteration
  • Haiku — fast, lightweight tasks & high-volume automation

/clear

Wipe the conversation and start fresh. Use between features to keep context clean.

/compact

Manually summarize the session to free up context space before hitting the limit. Use before auto-compact kicks in, or start a new chat when finishing a feature.

/mcp

Manage your MCP servers (plugins) — connect Figma, GitHub, browser control, and more.

/plugin

Install and manage Claude Code plugins — extend your agent with custom tools and integrations.

/context

Shows a snapshot of what's currently loaded into your conversation's context window — everything consuming tokens right now.

Bonus Tip

Stop Clicking "Accept"

Open Explorer, navigate to the .claude folder, open settings.local.json, and paste in the following permissions.

{
  "permissions": {
    "allow": [
      "Bash",
      "Read",
      "Edit",
      "Write",
      "MultiEdit",
      "Grep", "Glob", "LS",
      "WebFetch",
      "WebSearch",
      "NotebookRead",
      "NotebookEdit",
      "TodoRead", "TodoWrite"
    ]
  }
}
Highlighted = common requests from Claude. Bash (shell commands), Edit (file diffs), Write (create/overwrite), MultiEdit (batch edits), WebFetch (pull from URL), WebSearch (search the web).
Step 8

Edit Out the AI Aesthetic

Make It Yours

After your agent builds, go back and strip out the signs of AI-generated design. Copy the list to the right into your chat and double check. Also put the list in your PRD so it knows from the beginning.

Things That Look Like AI

  • Beveled corners on everything
  • Default AI fonts (Inter, system defaults)
  • Left-side menu bars
  • Cookie-cutter layouts
  • Generic gradient choices
  • Affordance design that glows
  • Overly polished glass/blur effects
  • Centered hero sections with stock phrases
  • Floating cards with drop shadows
  • Icon-heavy minimal dashboards
