Step 7

Let Your Agent Build

Best practices for prompting your agent. Hover each card for details.

Pseudo Code

Remember this concept and use it when you describe what you want.

A simplified, human-readable description of what your program should do — written in plain language, not actual code. It lets you think through logic without worrying about syntax.

“When the user clicks the canvas, generate a new shape at that position. Each shape should inherit the color of the previous one but rotate 15 degrees.”

Be Descriptive

The more detail you give, the better the output.

Instead of “make it look nice,” say: “Use a dark background (#0a0a0a), thin white lines, and animate them with a sine wave that oscillates every 3 seconds.”

Break the project into components: layout, interaction, animation, color, typography. Describe each one separately.

Think Like a Composer

Each chat is a different musician in your orchestra.

Every musician in an orchestra has a strength. Treat each chat the same way — one for planning, one for building, one for troubleshooting, one for editing, one for critique. Don’t cross-pollinate.

Step 7 — Continued

Let Your Agent Build

Supercharge your agent with Skills

What Are Skills?

Skills are reusable prompt files that give your agent specialized knowledge and capabilities. Install a skill once, then invoke it with a slash command — like giving your agent a new superpower on demand.

A Skill is a markdown file (stored in .claude/skills/) that contains expert-level instructions for a specific domain. When you type its slash command, the skill’s full prompt loads into the conversation — turning your general-purpose agent into a specialist.

Skills can be shared, versioned, and stacked. Think of them as plug-in expertise.

UI/UX Pro Max

Design intelligence for building professional UI/UX across multiple platforms.

An AI skill that gives your agent deep knowledge of UI/UX best practices — layout, typography, color theory, accessibility, responsive design, and platform conventions. It turns your coding agent into a design-aware builder.

github.com/nextlevelbuilder/ui-ux-pro-max-skill

Skill Creator

Anthropic’s official skill for building, testing, and iterating on new skills.

Invoke with /skill-creator — it has four modes: Create, Eval, Improve, and Benchmark. Describe what you want the skill to do and it generates the prompt file, runs evals, and iterates until it’s production-ready.

Step 7 — Continued

Let Your Agent Build

Useful /commands to know about

/model

Switch models mid-session. Use Opus for hard problems, Haiku for fast simple tasks.

Not every task needs the most powerful model. Match the model to the job:

Opus — complex reasoning, architecture decisions, hard bugs

Sonnet — everyday building, writing, iteration

Haiku — fast, lightweight tasks: quick edits, simple lookups, high-volume automation where speed and cost matter

/clear

Wipe the conversation and start fresh. Use between features to keep context clean.

/compact

Manually summarize the session to free up context space before hitting the limit.

When you hit the context limit, the agent auto-compacts: it summarizes earlier work to free up space but loses detail. Use /compact manually before that happens, or start a new chat when finishing a feature.

/mcp

Manage your MCP servers (plugins) — connect Figma, GitHub, browser control, and more.

/plugin

Install and manage Claude Code plugins — extend your agent with custom tools and integrations.

/context

Shows a snapshot of what’s currently loaded into your conversation’s context window — everything consuming tokens right now.

Stop Clicking “Accept”

Open Explorer, navigate to the .claude folder, open settings.local.json, and paste in the following permissions.

{
  "permissions": {
    "allow": [
      "Bash",
      "Read",
      "Edit",
      "Write",
      "MultiEdit",
      "Grep", "Glob", "LS",
      "WebFetch",
      "WebSearch",
      "NotebookRead",
      "NotebookEdit",
      "TodoRead", "TodoWrite"
    ]
  }
}

Step 8

Edit Out the AI Aesthetic

Make It Yours

After your agent builds, go back and strip out the signs of AI-generated design.

Copy and paste the list to the right — or create your own updated version based on what you learned in the AI landscape phase — into the chat and double check that it avoided these designs.

Also put the list in your PRD so it knows not to use it from the beginning.

Things That Look Like AI

  • Beveled corners on everything
  • Default AI fonts (Inter, system defaults)
  • Left-side menu bars
  • Cookie-cutter layouts
  • Generic gradient choices
  • Affordance design that glows
  • Overly polished glass/blur effects
  • Centered hero sections with stock phrases
  • Floating cards with drop shadows
  • Icon-heavy minimal dashboards

Example: openclaw.ai
